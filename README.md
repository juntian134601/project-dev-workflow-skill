# project-dev-workflow-skill

AI 辅助项目开发的结构化工作流程 Skill。

## 功能介绍

将项目开发分为 **7 个有序步骤**，确保 AI 辅助开发有条不紊地进行，避免过早编码和非受控重构。

```
Step1: 理解需求 → Step2: 任务分解
    ↓
Step3: 分析项目 → Step4: 编写 Spec
    ↓
Step5: 评审 Spec → Step6: 实施计划
    ↓
Step7: 增量编码
```

## 7 步工作流程

| 步骤 | 阶段 | 核心内容 | 关键约束 |
|------|------|----------|----------|
| Step1 | 理解需求 | 澄清业务目标、用户、范围、约束 | 只问"做什么"，不谈"怎么做" |
| Step2 | 任务分解 | 拆模块、定边界、排依赖、估复杂度 | 保持高层，不设计 API |
| Step3 | **项目分析** | 技术栈、目录结构、状态管理、API 层、UI 体系、权限、路由、构建、代码规范 | **只读不写，绝不改代码** |
| Step4 | **编写 Spec** | 功能目标、用户角色、权限模型、页面结构、API 设计、数据结构、路由权限、风险点、实施步骤 | **只写文档，绝不编码** |
| Step5 | 评审 Spec | 用户确认、补全边界、排查冲突 | 未确认不进入下一步 |
| Step6 | **实施计划** | 分阶段、列文件、标风险、标影响范围 | **只计划，不编码** |
| Step7 | **增量编码** | 按阶段实施，每步解释说明 | **每次 ≤5 文件、不重构、不改无关模块** |

## 触发关键词

以下关键词会触发此 Skill：

- `帮我开发这个项目`
- `开始一个新功能`
- `按流程开发`
- `分析项目并编写 spec`
- `AI 开发流程`
- `结构化开发`

## 安装方式

### 方式一：从 Release 下载（推荐）

1. 前往 [Releases](https://github.com/juntian134601/project-dev-workflow-skill/releases) 页面
2. 下载最新版本的 `project-dev-workflow.zip`
3. 解压到 `~/.workbuddy/skills/` 目录
4. 重启 WorkBuddy，Skill 自动生效

## 使用场景

### 场景 A：在现有项目中添加新功能

```
用户："帮我添加一个 RBAC 权限系统"
→ 执行 Step3（分析项目）
→ 执行 Step4（编写 Spec）
→ 执行 Step5（评审 Spec）
→ 执行 Step6（实施计划）
→ 执行 Step7（增量编码）
```

### 场景 B：从零开始开发新项目

```
用户："帮我从零开发一个 XX 系统"
→ 执行 Step1（理解需求）
→ 执行 Step2（任务分解）
→ 跳过 Step3（无现有项目）
→ 执行 Step4（编写 Spec）
→ ...
```

### 场景 C：快速修复/小改动

```
用户："修复这个 bug"
→ 执行 Step3-lite（针对性读取项目上下文）
→ 直接进入 Step7（聚焦修改）
```

## 核心原则

1. **先分析再设计** — 没有理解代码库之前绝不写 Spec
2. **先设计再编码** — 没有书面 Spec 之前绝不写代码
3. **先计划再执行** — 没有实施计划之前绝不开始编码
4. **小批量修改** — 一次不超过 5 个文件
5. **最小惊扰** — 绝不重构或修改无关代码
6. **全程解释** — 每个操作前后都必须说明
